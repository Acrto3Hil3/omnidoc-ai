# 📋 Plans

This folder holds **implementation plans** for OmniDoc AI. The working rule for this
project is **plan first, then code**: before any non-trivial feature or change, we
write a short plan here, agree on it, and only then implement against it.

## Why

- A plan forces the design decisions to surface *before* code is written.
- It gives a clear, reviewable record of *what* we're building and *why*.
- It makes it easy to pick work back up later, or hand it off.

## How it works

1. Copy [`TEMPLATE.md`](TEMPLATE.md) to a new file.
2. Name it `NNN-short-title.md` — a zero-padded sequence number + kebab-case title.
   Example: `001-document-upload.md`, `002-ai-qa-panel.md`.
3. Fill it in, discuss/adjust, then implement.
4. Keep the plan updated as reality diverges — a stale plan is worse than none.
   Mark the status field (`Draft` → `Approved` → `In progress` → `Done`).
